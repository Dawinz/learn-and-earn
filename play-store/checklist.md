# Play Store Checklist\n\nUse this as your master list. Check each box before proceeding to production rollout.\n\n## Pre-listing\n- [ ] Reserve app name in Play Console\n- [ ] Create app: default language, app name, package `com.learnandearn.app` (confirm)\n- [ ] Add testers (internal/closed) and upload contact details (see contacts.md)\n- [ ] Set up App access instructions if login is required\n\n## Store listing\n- [ ] App icon prepared (512x512 PNG, <1MB)\n- [ ] Feature graphic prepared (1024x500 JPG/PNG, <1MB)\n- [ ] Phone screenshots (min 2, up to 8; 1080x1920 or higher)\n- [ ] 7-inch and 10-inch tablet screenshots (optional but recommended)\n- [ ] Short description (80 chars)\n- [ ] Full description (4000 chars)\n- [ ] Category: Application > Education (confirm)\n- [ ] Content rating questionnaire completed\n- [ ] Data safety form completed\n- [ ] Privacy policy URL added and accessible (see privacy-policy.md)\n\n## App content\n- [ ] Ads declaration (uses Google Mobile Ads if applicable)\n- [ ] Target audience and content (13+ if rewards; confirm)\n- [ ] News apps declaration (if applicable)\n- [ ] Government apps declaration (if applicable)\n\n## Signing and builds\n- [ ] App signing by Google Play enabled OR upload signing key\n- [ ] Upload key created and safely stored\n- [ ] AAB built in release mode with versionCode bump\n- [ ] Proguard/R8 configured if using minify\n\n## Testing\n- [ ] Upload to Internal testing track; start a release\n- [ ] Share opt-in link with testers; collect feedback\n- [ ] Review Pre-launch report (devices, crashes, permissions)\n- [ ] Fix issues and iterate\n\n## Production rollout\n- [ ] Prepare release notes (see release-notes.md)\n- [ ] Start staged rollout (e.g., 5% → 20% → 50% → 100%)\n- [ ] Monitor ANRs/crashes and vitals\n- [ ] Respond to reviews and update listing if needed\n
